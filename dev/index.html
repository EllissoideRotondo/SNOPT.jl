SNOPT.jl

SNOPT.jl is an unofficial Julia wrapper for SNOPT (Sparse Nonlinear OPTimizer), a sequential quadratic programming (SQP) solver for smooth, large-scale, sparsely constrained nonlinear optimization problems of the form

\[\min_{x \in \mathbb{R}^n} \; f(x) \quad \text{subject to} \quad l \le \begin{pmatrix} x \\ c(x) \end{pmatrix} \le u,\]

where $f$ and the constraint functions $c$ are smooth and may be nonlinear.

The package exposes all three SNOPT Fortran entry points — snOptA, snOptB, and snOptC — through Julia callbacks, and adds a single high-level snopt function that covers the common case (objective + gradient, optional sparse nonlinear constraints).

Commercial solver required

SNOPT itself is closed-source commercial software. You must obtain a SNOPT license and a built libsnopt7 shared library separately; it is not bundled with this package. See Installation.

Concurrency

SNOPT.jl supports one active SNOPT solve at a time per Julia process. Sequential solves are supported, but concurrent solves from multiple Julia threads are not. Use separate Julia processes for parallel independent solves.

Quick start

using SNOPT

result = snopt(
    x -> (x[1] - 1)^2 + (x[2] - 2)^2,                          # objective f(x)
    (g, x) -> (g[1] = 2(x[1]-1); g[2] = 2(x[2]-2); nothing),   # gradient g!(g, x)
    [0.0, 0.0];                                                # starting point
    lb = -10.0, ub = 10.0,
    options = ["Major print level" => 0],
)

result.status_symbol   # :Solve_Succeeded
result.x               # ≈ [1.0, 2.0]
result.objective       # ≈ 0.0

Relationship to Optimization.jl

For modeling-first workflows, SNOPT is intended to be used through Optimization.jl via the OptimizationSNOPT.jl adapter, which handles automatic differentiation and problem assembly. That adapter is a work in progress; see Optimization.jl integration.
